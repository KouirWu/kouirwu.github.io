Titanic 生存预测学习报告

1. 项目概述

本项目旨在使用泰坦尼克号乘客数据，通过数据分析和机器学习技术，预测乘客的生存情况。主要步骤包括数据加载、数据探索与预处理、特征工程、模型选择与训练、以及结果评估与提交。

2. 数据探索与预处理

2.1 导入库与加载数据

首先，导入必要的Python库，包括pandas用于数据处理，numpy用于数值计算，seaborn和matplotlib用于数据可视化，以及sklearn中的各种机器学习算法。

加载训练集 (train.csv) 和测试集 (test.csv)，并将它们合并以便进行统一的数据预处理。

2.2 数据初步观察

查看数据集的特征名称、前几行和后几行数据，了解数据结构。

通过 info() 和 isnull().sum() 查看数据类型和缺失值情况。

Cabin 特征缺失值较多，Age 和 Embarked 也有部分缺失。

使用 describe() 查看数值型和类别型特征的统计信息。

2.3 特征与生存率分析

分析各个特征与生存率 (Survived) 之间的关系。

  • Pclass (船票等级): 等级越高（数值越小），生存率越高。
  • Sex (性别): 女性的生存率远高于男性。
  • SibSp (兄弟姐妹/配偶数量) 和 Parch (父母/子女数量): 这些特征与生存率的关系不是简单的线性关系，后续可以组合成家庭大小特征。

2.4 数据可视化分析

使用图表更直观地展示数据特征与生存率的关系。

  • 年龄 (Age) 与生存率的关系：不同年龄段的生存情况有所不同。
  • 船票等级 (Pclass)、年龄 (Age) 与生存率的关系：不同船票等级下，年龄与生存率的分布不同。
  • 登船港口 (Embarked)、船票等级 (Pclass)、性别 (Sex) 与生存率的关系。
  • 登船港口 (Embarked)、性别 (Sex)、票价 (Fare) 与生存率的关系。

2.5 数据清洗与特征工程

2.5.1 删除无关特征

删除 Ticket 和 Cabin 特征，因为 Ticket 较为杂乱，Cabin 缺失值过多。

2.5.2 创建 Title 特征

从 Name 特征中提取乘客的称谓 (Title)，并进行归类和数值化。

删除 Name 和 PassengerId (训练集)。

2.5.3 转换 Sex 特征

将 Sex 特征数值化：female -> 1, male -> 0。

2.5.4 填充 Age 缺失值并分箱

根据 Sex 和 Pclass 的组合，使用年龄中位数填充 Age 的缺失值。

将 Age 进行分箱处理，转换为序数特征。

2.5.5 创建 FamilySize 和 IsAlone 特征

合并 SibSp 和 Parch 创建 FamilySize 特征，并由此创建 IsAlone 特征。

删除 Parch, SibSp, FamilySize。

2.5.6 创建 Age*Pclass 交互特征

2.5.7 填充 Embarked 缺失值并转换

使用众数填充 Embarked 的缺失值，并将其转换为序数特征。

2.5.8 填充 Fare 缺失值并分箱

使用中位数填充测试集中 Fare 的缺失值，并对 Fare 进行分箱处理。

3. 模型选择与训练

准备训练数据和测试数据。

尝试了以下几种模型：

3.1 逻辑回归 (Logistic Regression)

3.2 决策树 (Decision Tree)

3.3 随机森林 (Random Forest)

4. 结果与提交

比较不同模型的准确率。

根据模型预测结果（此处以随机森林为例，因其在训练集上表现较好，但需注意过拟合风险），生成提交文件。

5. 总结与展望

5.1 总结

本项目通过对泰坦尼克号乘客数据的分析和建模，实现了对乘客生存情况的预测。主要工作包括：

  • 数据清洗与预处理：处理了缺失值，转换了数据类型。
  • 特征工程：创建了如 Title, FamilySize, IsAlone, Age*Pclass 等新特征，并对部分特征进行了分箱和数值化处理。
  • 数据可视化：通过图表分析了各特征与生存率的关系。
  • 模型训练与评估：尝试了逻辑回归、决策树和随机森林模型，并比较了它们在训练集上的准确率。

关键发现：

  • Sex (女性生存率更高) 和 Pclass (高等级船舱生存率更高) 是非常重要的预测特征。
  • Title 特征的引入对模型有积极影响。
  • 家庭大小 (FamilySize) 对生存率有一定影响，独自一人 (IsAlone) 的生存率相对较低。

5.2 展望与改进

  • 更精细的特征工程：可以尝试更多特征组合或转换方式，例如对 Fare 进行更细致的分箱或标准化。
  • 超参数调优：对选定的模型（如随机森林）进行超参数调优，以提高泛化能力。
  • 集成学习：尝试更高级的集成学习方法，如梯度提升树 (Gradient Boosting) 或 XGBoost。
  • 交叉验证：使用交叉验证来更准确地评估模型性能，避免过拟合。
  • 处理数据不平衡：如果目标变量存在类别不平衡，可以考虑使用过采样或欠采样技术。
  • 模型解释性：进一步分析模型（如随机森林的特征重要性）以更好地理解预测结果。
