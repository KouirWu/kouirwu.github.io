📜 283. 移动零

🔗 题目链接：283. 移动零

🎯 题目描述

给定一个数组 nums，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。

注意：必须在不复制数组的情况下原地对数组进行操作。

📋 约束条件

  • 1 <= nums.length <= 10^4
  • -2^31 <= nums[i] <= 2^31 - 1

💡 解题思路

核心思想：

  • 🎯 双指针法：利用两个指针，一个指向当前已处理好的非零序列的尾部（left 或 i0），另一个指针（right 或 i）遍历整个数组。
  • 🧹 原地操作：当遍历指针遇到非零元素时，将其与 left 指针指向的位置交换（或直接赋值到 left 指针位置并将原位置赋零），然后 left 指针向后移动。
  • 🔄 保持相对顺序：由于非零元素是按顺序被放到 left 指针及其之前的位置，所以它们的相对顺序得以保持。

步骤（以方法一为例）：

  1. 初始化指针 👈👉：left = 0 (指向下一个非零元素应放置的位置)，right = 0 (当前遍历到的元素位置)。
  2. 遍历数组 🚶‍♂️：当 right 指针未越界时：
    • 检查元素 🤔：如果 nums[right] 不为 0：
      • 交换元素 🔄：将 nums[left] 和 nums[right] 的值交换。
      • 移动 left 指针 ➡️：left += 1。
    • 移动 right 指针 ➡️：right += 1。
  3. 完成 🎉：遍历结束后，所有非零元素都在数组前面，且保持相对顺序，零元素则被移到了后面。

📝 代码逐行解析 (以方法二为例)

  1. i0 = 0

    • 功能：初始化一个指针 i0。这个指针的作用是标记数组中所有非零元素已经排好序的区间的下一个位置。也就是说，nums[0...i0-1] 中的所有元素都是非零的，并且是按照它们在原数组中的相对顺序排列的。
    • 用途：i0 指向的是下一个非零元素应该被放置的目标索引。

  2. for i in range(len(nums)):

    • 功能：遍历输入数组 nums 中的每一个元素。i 是当前元素的索引。

  3. if nums[i]:

    • 功能：检查当前遍历到的元素 nums[i] 是否为非零。在 Python 中，整数 0 在布尔上下文中被视为 False，任何非零整数都被视为 True。
    • 逻辑：如果 nums[i] 是一个非零元素，那么它需要被移动到 i0 指针所指向的位置。

  4. nums[i0], nums[i] = nums[i], nums[i0]

    • 功能：这是一个 Python 特有的元组赋值语法，用于交换两个变量的值。这里它交换了 nums[i0] 和 nums[i] 的值。
    • 效果：
      • 如果 i 和 i0 指向同一个位置（即 nums[i] 本身就是一个在其正确位置的非零元素，且其前面没有0），那么这个交换是原地交换，没有实际效果，但逻辑上正确。
      • 如果 nums[i] 是一个非零元素，而 nums[i0] 是一个零（因为 i0 之前的都是非零，i0 可能是第一个零的位置，或者 i0 追上了 i），那么这个操作会将非零元素 nums[i] 移动到 i0 位置，并将原来的 nums[i0]（可能是0）移动到 i 位置。

  5. i0 += 1

    • 功能：在成功将一个非零元素放置到 nums[i0] 位置后，i0 指针向右移动一位。
    • 意义：这表示 nums[0...i0-1] 这个区间的非零元素又增加了一个，i0 继续指向下一个非零元素应该被放置的目标位置。

⏱️ 时间复杂度分析

  • 遍历数组 🚶‍♂️ → O(N)
    • N 是 nums 数组中元素的数量。
    • 两种方法都只需要单次遍历数组。
  • 元素交换 🔄 → O(1)
    • 每次交换操作是常数时间。
  • 总时间复杂度：O(N)，因为主要操作是遍历数组。

空间复杂度分析

  • 额外空间 💾 → O(1)
    • 两种方法都只使用了有限的几个变量（指针）来辅助操作，没有使用与输入数组大小相关的额外空间。
    • 操作是在原数组上直接进行的（原地算法）。
  • 总空间复杂度：O(1)。

🆚 对比两种双指针方法

特点 方法一 (while循环) 方法二 (for循环)
指针含义 left: 非零区尾部, right: 扫描指针 i0: 非零区尾部, i: 扫描指针
代码结构 显式的 while 循环和 right 指针递增 更简洁的 for 循环，隐式 i 指针递增
核心判断 if nums[right] != 0: if nums[i]: (利用Python真值特性)
交换逻辑 明确的 nums[left], nums[right] = ... 类似的 nums[i0], nums[i] = ...
可读性 传统双指针，逻辑直观 更 Pythonic，对新手可能需理解真值判断
性能 理论上相同 理论上相同

两种方法在本质上都是双指针思想的体现，核心都是维护一个非零元素区，并将遍历到的非零元素填充到这个区域的末尾。方法二因为利用了 Python 的特性，代码更为凝练。

📜 总结

  1. 核心技巧 🔑：双指针法是解决这类原地修改数组、移动特定元素的经典且高效的策略。
  2. 原地操作 ⚙️：通过在原数组上进行元素交换，实现了 O(1) 的空间复杂度，满足题目要求。
  3. 保持相对顺序 📏：算法的设计确保了非零元素在移动过程中的相对顺序不会被打乱。
  4. 两种实现 ✌️：无论是 while 循环还是 for 循环版本，核心思想一致，for 循环版本在 Python 中更为简洁。
  5. 边界情况 🧭：算法能正确处理空数组、全零数组、全非零数组等边界情况。

🚀 练习建议：

  • 尝试思考如果题目要求不保持非零元素的相对顺序，是否有更简单的解法？（提示：可以将所有非零元素移到前面，然后填充0）
  • 将此双指针思想应用到其他类似问题，如 LeetCode 27. 移除元素。
