📜 128. 最长连续序列

🔗 题目链接：128. 最长连续序列

🎯 题目描述

给定一个未排序的整数数组 nums，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

📋 约束条件

  • 0 <= nums.length <= 10^5
  • -10^9 <= nums[i] <= 10^9

💡 解题思路

核心思想：

  • 🎯 利用哈希表（集合）快速查找元素
  • 🔍 只从连续序列的起始点开始计算长度
  • ⚡ 避免重复计算，确保 O(n) 时间复杂度

算法步骤：

  1. 📖 将所有数字存入集合 st，实现 O(1) 的查找时间
  2. 🔄 遍历集合中的每个数字 i：
    • 🔑 检查 i-1 是否在集合中
    • ❌ 如果 i-1 存在，说明 i 不是序列起点，跳过
    • ✅ 如果 i-1 不存在，说明 i 是序列起点
  3. 📏 从起点开始计算连续序列长度：
    • 🔢 从 i+1 开始，依次检查后续数字是否存在
    • 📈 记录当前序列长度，更新最大长度

💡 代码详细解释

逐行解析：

  1. ans = 0

    • 功能：初始化答案变量，用于记录最长连续序列的长度。

  2. st = set(nums)

    • 功能：将输入数组转换为集合。
    • 优势：
      • 自动去除重复元素
      • 提供 O(1) 的查找时间复杂度
      • 比在列表中查找的 O(n) 时间复杂度更高效

  3. for i in st:

    • 功能：遍历集合中的每个唯一数字。

  4. if i - 1 in st: continue

    • 核心优化：这是算法的关键所在！
    • 逻辑：如果 i-1 存在于集合中，说明 i 不是连续序列的起始点。
    • 为什么跳过：避免重复计算。例如，对于序列 [1,2,3,4]：
      • 当遍历到 2 时，发现 1 存在，跳过
      • 当遍历到 3 时，发现 2 存在，跳过
      • 当遍历到 4 时，发现 3 存在，跳过
      • 只有遍历到 1 时，发现 0 不存在，才开始计算
    • 时间复杂度保证：确保每个数字最多被访问两次（一次在外层循环，一次在内层while循环）。

  5. j = i + 1

    • 功能：从当前数字的下一个数字开始检查连续性。

  6. while j in st: j += 1

    • 功能：持续检查后续数字是否存在，直到找到不连续的位置。
    • 示例：如果 i=1，会依次检查 2, 3, 4… 直到某个数字不存在。

  7. ans = max(ans, j - i)

    • 功能：更新最长序列长度。
    • 计算：j - i 表示从 i 开始的连续序列长度。
    • 示例：如果 i=1，j=5，则序列长度为 5-1=4，对应 [1,2,3,4]。

⏱️ 时间复杂度分析

  • 集合构建 🏗️ → O(n)：遍历数组一次
  • 外层循环 🔄 → O(n)：遍历集合中的每个元素
  • 内层while循环 🔍 → 总共 O(n)：
    • 虽然是嵌套循环，但每个数字最多被访问一次
    • 关键在于 if i - 1 in st: continue 的优化
    • 确保只从序列起点开始计算
  • 总时间复杂度：O(n) ✅

空间复杂度：O(n) - 存储集合所需的空间

🎯 关键要点总结

  1. 核心技巧：只从连续序列的起始点开始计算，避免重复计算
  2. 时间复杂度保证：通过 if i - 1 in st: continue 确保每个数字最多被访问两次
  3. 空间换时间：使用集合实现 O(1) 查找，提升整体效率
  4. 边界处理：正确处理空数组和单元素数组的情况
  5. 面试重点：这是一道经典的哈希表应用题，考察对时间复杂度优化的理解
