🔍 1. 两数之和

题目链接：1. 两数之和

📝 题目描述

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 target 的那 两个 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。

你可以按任意顺序返回答案。

题目要求 🎯

给定一个整数数组 nums 和一个目标值 target，找出数组中和为目标值的两个数的下标。

🧠 解题思路

这道题可以用两种主要方法解决：暴力解法和哈希表解法。

方法一：暴力解法 🔄

核心思想：

  • 遍历所有可能的数对组合，检查它们的和是否等于目标值
  • 使用两层嵌套循环，外层循环选择第一个数，内层循环选择第二个数

步骤：

  1. 使用两层循环遍历数组
    • 外层循环 i 从 0 到 n-2
    • 内层循环 j 从 i+1 到 n-1
  2. 检查每对数字之和是否等于目标值
    • 如果 nums[i] + nums[j] == target，返回 [i, j]

💻 代码详细解释

方法一：暴力解法 🔄

逐行解析：

  1. n = len(nums) 📏

    • 获取数组的长度，用于循环控制。

  2. 两层循环 🔄

    • 外层循环 for i in range(n) 选择第一个数
    • 内层循环 for j in range(i + 1, n) 选择第二个数
    • 注意内层循环从 i+1 开始，避免重复使用同一个元素

  3. if nums[i] + nums[j] == target ✅

    • 检查当前两个数的和是否等于目标值
    • 如果相等，直接返回这两个数的下标 [i, j]

方法二：哈希表解法 🗃️

逐行解析：

  1. hash_map = {} 🗃️

    • 初始化一个空字典（哈希表），用于存储已遍历过的数字（作为键）及其下标（作为值）

  2. for i, num in enumerate(nums) 🔄

    • 使用 enumerate() 同时获取数组元素及其下标
    • i 是下标，num 是对应的数字

  3. complement = target - num 🧮

    • 计算当前数字 num 需要的配对数字
    • 如果存在一个数 complement 使得 num + complement = target，则找到了答案

  4. if complement in hash_map 🔍

    • 检查配对数字是否已经在哈希表中
    • 如果在，说明之前已经遍历过这个配对数字，直接返回两个数的下标

  5. hash_map[num] = i 📝

    • 如果没找到配对，将当前数字及其下标存入哈希表，继续遍历

⏱️ 时间复杂度分析

方法一：暴力解法 🔄

  • 时间复杂度：O(n²)，其中 n 是数组长度
    • 需要两层循环遍历所有可能的数对
  • 空间复杂度：O(1)，只需要常数级别的额外空间

方法二：哈希表解法 🗃️

  • 时间复杂度：O(n)，其中 n 是数组长度
    • 只需要遍历数组一次
    • 哈希表的查找操作平均时间复杂度为 O(1)
  • 空间复杂度：O(n)
    • 需要一个哈希表存储已遍历的数字，最坏情况下需要存储 n-1 个元素

📊 方法对比

方法 时间复杂度 空间复杂度 优缺点
暴力解法 🔄 O(n²) O(1) 简单直观，但效率低
哈希表解法 🗃️ O(n) O(n) 效率高，是最优解

🔍 代码运行示例

以哈希表解法为例，演示解题过程：

输入：nums = [2, 7, 11, 15], target = 9

执行过程：

迭代 i num complement hash_map 结果
1 0 2 7 {} 继续
2 1 7 2 {2: 0} 返回 [0, 1]

解释：

  1. 第一次迭代：

    • 当前数字 num = 2，需要的配对数字 complement = 9 - 2 = 7
    • hash_map 为空，没有找到配对数字
    • 将 2 及其下标 0 存入哈希表：hash_map = {2: 0}

  2. 第二次迭代：

    • 当前数字 num = 7，需要的配对数字 complement = 9 - 7 = 2
    • 检查 2 是否在 hash_map 中，发现存在
    • 返回 [hash_map[2], i] 即 [0, 1]

📝 总结

  1. 两数之和是一道经典的数组和哈希表题目，也是 LeetCode 的第一题。
  2. 暴力解法简单直观，但时间复杂度为 O(n²)，不够高效。
  3. 哈希表解法通过空间换时间的策略，将时间复杂度降低到 O(n)，是解决此类问题的常用技巧。
  4. 这道题的核心思想是：对于每个数 x，检查 target - x 是否存在于数组中。
  5. 哈希表的应用是解决此类查找问题的有力工具，值得掌握。

✅ 练习建议：尝试解决类似的变种问题，如三数之和、四数之和，加深对哈希表应用的理解！
