📜 49. 字母异位词分组

🔗 题目链接：49. 字母异位词分组

🎯 题目描述

给你一个字符串数组 strs，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。

字母异位词 是由重新排列源单词的所有字母得到的一个新单词。

📋 题目要求

给定一个字符串数组 strs，将字母异位词（Anagram）组合在一起，并返回这些组合后的列表。

🔑 字母异位词（Anagram）定义

  • 定义：两个字符串如果包含相同的字母（顺序无关），则互为字母异位词。
  • 示例：
    • "eat" 和 "tea" 是字母异位词，因为它们的字母顺序不同但组成相同。
    • "rat" 和 "bat" 不是字母异位词，因为它们包含的字母不同。

📥 输入输出示例

输入：strs = ["eat", "tea", "tan", "ate", "nat", "bat"]
输出：

💡 解题思路

核心思想：

  • 🎯 字母异位词在排序后会变成相同的字符串

    • "eat"、"tea"、"ate" 排序后都变成 "aet"
    • "tan"、"nat" 排序后变成 "ant"
    • "bat" 排序后变成 "abt"

  • 🗂️ 利用哈希表（字典）分组存储：

    • 使用 排序后的字符串 作为 key
    • 相同 key 的字符串分组存储

步骤：

  1. 初始化一个 defaultdict(list) 📖，用于存储分组结果。
    (defaultdict 可以避免 KeyError，如果 key 不存在会自动创建空列表)
  2. 遍历每个字符串 s 🔄：
    • 排序 s 得到一个标准化的 key 🔑
    • 把 s 添加到 d[key] 的分组里 ➕
  3. 最后返回字典的所有 values() 📤，就是分组后的结果。

💻 代码详细解释

逐行解析：

  1. d = defaultdict(list)

    • 功能：初始化一个特殊的字典 d。如果尝试访问一个不存在的键，它会自动为该键创建一个空列表 [] 作为值。
    • 用途：键 (key) 将是 排序后的字符串 (例如："aet")，值 (value) 是一个 列表 (例如：["eat", "tea", "ate"])，用于存储所有排序后与键相同的原始字符串。

  2. for s in strs:

    • 功能：遍历输入列表 strs 中的每一个原始字符串 s。

  3. sorted_s = ''.join(sorted(s))

    • sorted(s)：将字符串 s 转换为一个其字符按字母顺序排列的列表。
      • 示例：sorted("tea") 结果是 ['a', 'e', 't']。
    • ''.join(...)：将这个排序后的字符列表重新连接成一个字符串。
      • 示例：''.join(['a', 'e', 't']) 结果是 "aet"。
    • 核心逻辑：所有字母异位词（如 "eat", "tea", "ate"）在经过此操作后都会得到相同的标准化字符串（"aet"）。这个标准化字符串 sorted_s 将用作哈希表的键。

  4. d[sorted_s].append(s)

    • 功能：将当前的原始字符串 s 添加到字典 d 中以 sorted_s 为键的列表里。
      • 如果键 sorted_s 在字典中已存在，则 s 被追加到其对应的列表中。
      • 如果键 sorted_s 不存在，defaultdict(list) 的特性会先为这个新键创建一个空列表，然后 s 被添加到这个新创建的列表中。

  5. return list(d.values())

    • d.values()：返回字典 d 中所有的值（即所有分组列表的集合）。在 Python 3 中，这返回一个视图对象（dict_values）。
    • list(...)：将这个视图对象转换为一个标准的列表。

⏱️ 时间复杂度分析

  • 排序每个字符串 🔄 → O(N * K log K)
    • N 是 strs 数组中字符串的数量。
    • K 是 strs 数组中字符串的最大长度。
    • 对每个字符串排序需要 O(K log K)。
  • 哈希表查存操作 🗃️ → O(N * K) (在最坏情况下，如果哈希冲突或者键的比较需要 O(K))
    • 遍历 N 个字符串，每个字符串作为键存入哈希表。
  • 总时间复杂度：主要是排序操作主导，因此是 O(N * K log K)。
    （这通常优于 O(N^2 * K) 的暴力两两比较方法）

🆚 对比其他方法

方法 时间复杂度 空间复杂度 适用场景
排序 + 哈希表 (本解法) O(N * K log K) O(N * K) ✅ 推荐，通用性好
暴力法 (两两比较) O(N^2 * K) O(N * K) ❌ 不推荐 (效率低下)
字符计数作为 key O(N * K) O(N * K) 🚀 字符集较小时更优

适用场景：如果字符串非常长（导致排序 K log K 显著大于计数 K），或者字符集固定且较小（如仅小写英文字母），此方法可能更优。

📜 总结

  1. 核心策略：排序 + 哈希表 是解决字母异位词分组问题的一种经典且高效的方法，时间复杂度为 O(N * K log K)。
  2. defaultdict(list)：这是一个非常实用的数据结构，可以简化分组逻辑，避免了在添加元素前检查键是否存在的繁琐步骤。
  3. 键的选择：将 排序后的字符串作为哈希表的 key 是关键，它确保了所有字母异位词都能映射到同一个键，从而被正确地归入同一组。
  4. 代码简洁性：这种方法的 Python 实现非常简洁明了，易于理解和掌握。
  5. 面试热门：这是一个常见的面试题目，考察对哈希表、字符串处理和时间复杂度分析的理解。
