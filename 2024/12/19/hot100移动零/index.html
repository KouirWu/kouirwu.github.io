LeetCode Hot 100 - 283. 移动零

2024-12-19
算法学习

题目描述

给定一个数组 nums，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。

注意：必须在不复制数组的情况下原地对数组进行操作。

约束条件

  • 1 <= nums.length <= 10^4
  • -2^31 <= nums[i] <= 2^31 - 1

算法思路

这道题可以使用双指针的方法来解决：

  1. 双指针设置：使用 left 和 right 两个指针，都从数组开头开始
  2. 移动策略：
    • right 指针遍历整个数组
    • 当 nums[right] 不为 0 时，将其与 nums[left] 交换
    • 交换后 left 指针向前移动一位
  3. 结果：遍历完成后，所有非零元素都被移到了前面，零元素自然就在后面了

时间复杂度：O(n) - 只需要遍历一次数组
空间复杂度：O(1) - 只使用了常数级别的额外空间

算法执行过程分析

以 nums = [0,1,0,3,12] 为例：

步骤 left right nums[right] 操作 数组状态
初始 0 0 0 - [0,1,0,3,12]
1 0 0 0 right++ [0,1,0,3,12]
2 0 1 1 交换,left++,right++ [1,0,0,3,12]
3 1 2 0 right++ [1,0,0,3,12]
4 1 3 3 交换,left++,right++ [1,3,0,0,12]
5 2 4 12 交换,left++,right++ [1,3,12,0,0]

关键点总结

  1. 双指针技巧：left 指针维护非零元素的边界，right 指针用于遍历
  2. 原地操作：通过交换元素位置，避免使用额外空间
  3. 相对顺序保持：由于 left 指针总是从左到右移动，保证了非零元素的相对顺序不变
  4. 边界处理：当数组全为零或全为非零时，算法依然正确工作

扩展思考

  • 进阶优化：可以减少不必要的交换操作（当 left == right 时不需要交换）
  • 类似问题：这种双指针技巧可以应用到其他”移动特定元素”的问题上
  • 稳定性：该算法保持了非零元素的相对顺序，是一个稳定的算法
